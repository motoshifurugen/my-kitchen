# 08: コアマイクロコピー
_Core Microcopy Specification_

---

## 概要

アプリ内で使用するコアマイクロコピー（空状態、エラー、記録完了）を定義する。
Voice & Toneガイドに基づき、一貫した言葉づかいを実現する。

**対象プラットフォーム:** iOS / Android（React Native）

**関連イシュー:** UX-060, UX-061, UX-062, UX-063

**準拠ドキュメント:**

- `02-core-wireframes.md` — ワイヤーフレーム・ラベル

## Voice & Tone

### Voice（声）— 5つの形容詞

1. **静か** — 情報量が少ない、余白がある
2. **やさしい** — 判断しない、責めない
3. **軽い** — 短い、押し付けない
4. **誠実** — 嘘をつかない、過剰に盛らない
5. **ほのかにワクワク** — 儀式の締めだけ少し明るい

### Tone（調子）— 5原則

1. **急かさない** — 「今すぐ」「早く」は使わない
2. **褒めすぎない** — 「最高！天才！」は避ける
3. **意味づけしない** — 人生論を語らない
4. **視線は"横"** — 上から指導しない、下から媚びない
5. **具体的で短い** — 情緒語は控えめ

### 敬語レベル

- 基本: **丁寧語（です・ます）**
- 硬すぎない短文中心
- タメ口・敬語過多は禁止

---

## 一貫性ルール（Consistency Rules）

### 行の長さ

| 種類 | 目安 |
|------|------|
| メッセージ本文 | 1〜2行（最大40文字程度） |
| ボタンラベル | 2〜6文字 |
| フッターラベル | 2文字 |

### 句読点・記号

| ルール | 説明 |
|--------|------|
| 「。」 | 文末に使用可。連続文では省略も可 |
| 「、」 | 控えめに。短文なら不要 |
| 「！」 | **原則使用しない**（セレブレーション含む） |
| 「？」 | 疑問形のみ。控えめに |
| 「…」 | 余韻として使用可。多用しない |

### 話者視点

アプリは**中立的な語り手**として話す。

| 方針 | 説明 |
|------|------|
| 一人称 | 使わない（「私は」「わたしたちは」禁止） |
| 二人称 | 控えめに（「あなた」は避ける。必要なら「お」をつけて丁寧に） |
| 主語省略 | 基本は主語なし。事実を静かに述べる |

**例:**
- ✅ 「記録しました。」
- ✅ 「棚に1枚、追加されました。」
- ❌ 「あなたの料理を記録しました！」
- ❌ 「私たちがお手伝いします」

## 推奨語彙 / 禁止語彙

### 望ましい語彙

```
静かに、そっと、今日、ひとつ、積み重なり、
追加、記録、余韻、片付け、残す、増える
```

### 禁止語彙

```
今すぐ、早く、急いで、最高、天才、神、
成功、勝ち、負け、変われる、成長できる、
がんばれ、完璧、すごい、やったね、
レベル、レベルアップ、評価、スコア、ポイント
```

## 画面別マイクロコピー

### S-02: アーカイブ（空状態）

```
まだ何もありません。

最初のひと皿を、
ここに残してみませんか。
```

**NG例**: 「棚が空っぽです！今すぐ記録しよう！」

### S-05: 探索（検索結果なし）

```
見つかりませんでした。

別の言葉で探してみてください。
```

**NG例**: 「残念！該当する料理がありません」

### S-03: カード詳細（写真なし）

```
写真はまだありません。

次に作るときに、追加できます。
```

**NG例**: 「写真がないと寂しいです！」

## 記録フロー（S-04）マイクロコピー

### S-04a: 写真撮影/選択

| 要素 | テキスト |
|------|---------|
| 画面タイトル | 記録する |
| メイン操作 | 写真を撮る |
| サブ操作 | ライブラリから選ぶ |
| スキップ | 写真なしで続ける |
| 補足テキスト | 写真はあとからでも追加できます |

**注:** 「写真なしで続ける」は保存ではなく次のステップ（S-04b）へ進む操作。

### S-04b: 入力フォーム

| 要素 | テキスト |
|------|---------|
| 画面タイトル | 記録する |
| 料理名ラベル | 料理名 * |
| 料理名プレースホルダー | （空欄） |
| メモラベル | メモ（任意） |
| タグラベル | タグ（任意） |
| タグ追加 | + タグを追加 |
| 次へボタン | 確認へ |

### S-04c: 確認・保存

| 要素 | テキスト |
|------|---------|
| 画面タイトル | 確認 |
| 補足テキスト | 記録はあとから編集できます |
| 保存ボタン | 保存する |

### S-08: セレブレーション

記録完了時に表示。ランダムで選択、連続で同じメッセージは避ける。

**バリエーション:**

1. ```記録しました。```
2. ```棚に1枚、追加されました。```
3. ```今日の分、残しておきます。```
4. ```また作った日が増えました。```
5. ```静かに、積み重なりました。```
6. ```今日のひと皿、残しました。```
7. ```棚に、ひとつ増えました。```

**NG例:**
```
やったね！最高！
完璧な記録です！
すごい！継続できてますね！
天才シェフ！
棚が育ちました！（評価的に聞こえる）
```

## エラーメッセージ

### 原則

- 責めない
- 急かさない
- 説明しすぎない
- 起きたことだけを静かに伝える

### 保存失敗

```
保存できませんでした。

もう一度お試しください。
```

### 読み込み失敗

```
読み込めませんでした。

少し待ってから、
もう一度お試しください。
```

### 写真アクセス失敗

```
写真にアクセスできませんでした。

設定から許可をご確認ください。
```

### ネットワークエラー（Phase 2）

**注:** MVP はローカルファースト。以下は将来のクラウド同期機能用。

```
接続できませんでした。

オフラインでも記録は残ります。
```

## 「AIっぽくない」チェックリスト

コピーを書いたら、以下を確認：

- [ ] 過度に丁寧すぎないか
- [ ] 「！」を使っていないか
- [ ] 定型文的な表現になっていないか
- [ ] 評価・判断を含んでいないか
- [ ] ユーザーを煽っていないか
- [ ] 長すぎないか（1〜2文が理想）
- [ ] 感情を押し付けていないか
- [ ] 「素晴らしい」「素敵」「すごい」等を使っていないか
- [ ] 命令形になっていないか
- [ ] 比較を促していないか
- [ ] ゲーム的な表現（レベル、スコア、達成）を使っていないか
