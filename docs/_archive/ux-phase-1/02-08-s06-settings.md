## S-06: 設定

### レイアウト構成

```
┌─────────────────────────────┐
│ [←]         設定            │ ← S-01 へ戻る
├─────────────────────────────┤
│                             │
│ ─ 通知 ─                    │
│   通知設定              [>] │
│                             │
│ ─ 音の演出 ─                │
│   音の演出            [ON]  │
│   ※ 無音でも、同じように使えます。 │
│                             │
│ ─ データ ─                  │
│   Kitchen Signals      [>]  │
│   データとプライバシー  [>] │ ← S-06a へ遷移
│                             │
│ ─ アプリについて ─          │
│   バージョン          1.0.0 │
│                             │
│  ℹ️ データは端末内に保存     │ ← 安心感の明記
│     されています            │
│                             │
├─────────────────────────────┤
│         （フッター非表示）    │ ← 設定画面はフッターなし
└─────────────────────────────┘
```

### 仕様

- **アクセス**: S-01 トップ画面のヘッダー（⚙️）からのみ
- **戻る**: ← ボタンで S-01 へ
- **フッター**: 非表示（頻度が低い画面のため）

### インタラクションゾーン

| ゾーン | タップ対象 | 期待結果 |
|--------|-----------|----------|
| 左上 | ← ボタン | S-01 へ戻る |
| 通知設定 | 行全体 | 通知設定詳細へ |
| 音の演出 | トグル | アプリ内の音演出を一括ON/OFF |
| Kitchen Signals | 行全体 | Signals編集画面へ |
| データとプライバシー | 行全体 | S-06a データとプライバシーへ |

### 制約（World Bible準拠）

- SNS連携、共有ボタンは存在しない
- アカウント登録は不要
- ローカルファースト明記

### ローカル通知（招待型） ※MVP

#### 目的

催促ではなく、「台所に戻るきっかけ」をそっと差し出す。未ログイン・未記録を責めない。

#### UI（MVPは1スイッチのみ）

- **項目名**：通知（夕方の招待）
- **操作**：ON / OFF のみ
- **初期値**：ON
- **補助文（任意）**：ℹ️ 夕方に、そっとお知らせします
- **OS通知許可が未取得の場合**：ONにしたタイミングで許可導線を表示（強い文言は避ける）

#### 送信ルール（固定）

- **送信**：1日1回まで
- **時刻**：毎日 17:30
- **条件**：その日に一度もログインしていない場合のみ送る
- **タップ時遷移**：S-01（キッチン世界）

#### 文言（決定・2パターン）

- **Pattern A**
  - タイトル：台所の本棚
  - 本文：今夜のためのページが、そっと開いています。
- **Pattern D**
  - タイトル：台所の本棚
  - 本文：今日のひと皿を、そっと置けます。

#### MVPでやらない

- 時刻変更 / 種別切替 / Quiet Hours / 曜日設定 / 複数通知

#### NG表現

- 「まだ」「未」「忘れ」など未達を指摘する語
- 命令調（〜しよう / 〜しなきゃ）
- 本文に「記録」「継続」など評価・自己啓発っぽい語を入れない

### 音の演出（更新版）

#### 目的

World Bible 準拠：
- 「音は主張しない。しかし、確かにそこに在る。」
- 「無音でも成立するが、無音より心地いい状態を目指す。」

#### 1) 設定項目（MVP）

##### 表示場所

S-06 設定画面の「音の演出」セクション。

##### UI（トグルのみ）

- **項目名**: 音の演出
- **操作**: ON/OFF

このトグルは以下をまとめて制御する（ユーザーには内訳を押し付けない）：
- キッチンの空気感を支える控えめなループ音（あれば）
- 記録完了などの小さな区切り音（あれば）

##### マイクロコピー（短文）

音の演出セクション末尾に小さく：
- 「無音でも、同じように使えます。」

#### 2) 音量調整UIの有無（結論）

**結論**: MVPでは音量スライダーは置かない

**理由**:
- 設定の複雑化を避ける（UX-009）
- システム音量に委ねるのが自然
- "主張しない音"は設計側のミックスで担保する

#### 3) デフォルト設定値（結論）

- **音の演出**: ON（控えめ）

##### 補足（驚かせない）

- 初回起動直後は鳴らさず、S-01表示後に数秒フェードイン、または初回タップ後にフェードインなどで「急に鳴る」を避ける
- OFFでも体験が破綻しないことが前提

### 設定項目最小化チェックリスト（UX-009）

以下を満たさない項目は追加しない：

- [ ] ユーザーが実際に変更する必要があるか
- [ ] 変更しないとアプリが使えない状況があるか
- [ ] World Bible の原則（プライバシー、シンプル、静か）に反していないか
- [ ] SNS連携、共有、アカウント登録を促していないか

**MVP で含めない設定:**
- アカウント登録/ログイン
- SNS連携
- データエクスポート（Phase 2）
- クラウド同期（Phase 2）
- テーマ選択（存在しない）

## S-06a: データとプライバシー

### 目的

ユーザーに「このアプリのデータは基本的に端末内にある」という安心感を与える。
同期や共有は前提にせず、ローカルファーストを明確にする。

### レイアウト構成

```
┌─────────────────────────────┐
│ [←]  データとプライバシー     │ ← S-06 へ戻る
├─────────────────────────────┤
│ データは端末内に保存されます  │
│ 写真やメモは基本この端末の中。 │
│ 初期状態では外部へ送信しません。│
│ 同期は将来"任意"（初期OFF）。  │
│                             │
│ ─ データ ─                  │
│ データを取り出す（将来）  [>] │
│                             │
│ ─ 同期（将来） ─            │
│ クラウド同期          [OFF]  │
│                             │
│ ─ 危険操作 ─                │
│ データを削除する        [>] │
└─────────────────────────────┘
```

### 仕様

- **アクセス**: S-06 設定画面の「データとプライバシー」行から
- **戻る**: ← ボタンで S-06 へ
- **フッター**: 非表示

### 1) データ保存場所の説明（必須）

#### 説明文（固定表示）

- **見出し**: データは端末内に保存されます
- **本文（短文・安心優先）**:
  - 「写真やメモは、基本的にこの端末の中に保存されます。」
  - 「アカウント登録はありません。初期状態では外部へ送信しません。」
  - 「クラウド同期は、将来機能として"任意"で追加予定です（初期はOFF）。」

**表現方針:**
- 「断定 + 例外（将来）を小さく」の形式
- "監視/追跡" を想起させる言い回しは避ける

### 2) データエクスポート UI（将来実装・UIのみ定義）

#### 表示

S-06a 内に「データを取り出す（エクスポート）」行を置く。

- **ラベル**: データを取り出す
- **サブテキスト**: （将来）
- **右側**: [>]

#### 挙動

- MVPでは未実装。タップするとモーダルで案内のみ表示。

#### 未実装モーダル文案

- **タイトル**: 準備中です
- **本文**: 「この機能は、あとから追加予定です。いまは端末内で静かに保存されます。」
- **ボタン**: 閉じる

### 3) データ削除 UI（MVPで実装する前提）

#### 表示

S-06a 内に「データを削除する」行を置く（危険操作なので最下部）。

- **ラベル**: データを削除する
- **サブテキスト**: この端末の記録を消去します
- **右側**: [>]

#### 削除フロー（S-06b: データ削除 確認モーダル）

1. **1段階目**: 説明 + 抑制（"責めない"・"戻れる"）
2. **2段階目**: 最終確認（文言入力 or 長押しのどちらか）

**推奨**: 文言入力なし、長押し方式（負荷が少ない）

- **タイトル**: データを削除しますか？
- **本文**:
  - 「この端末にある台所の記録がすべて消えます。」
  - 「この操作は取り消せません。」
- **ボタン**:
  - キャンセル
  - 長押しで削除（3秒）（危険色は使いすぎない・UIは静かに）

#### 削除完了トースト

- 「台所を静かに片づけました。」（※"削除しました"を避けてもOK）

### 4) クラウド同期 ON/OFF UI（将来実装・初期OFF）

#### 表示

S-06a 内に「クラウド同期」トグルを配置。

- **ラベル**: クラウド同期
- **初期状態**: OFF
- **サブテキスト**: （将来）端末の外にも保存できるようにする

#### 挙動

- MVPではトグルは表示するが操作不可（またはタップで未実装モーダル）
- もしくは「将来」セクションにまとめ、誤解を避ける

#### 未実装モーダル文案

- **タイトル**: 同期はまだありません
- **本文**: 「今は端末内だけで保存されます。同期は"任意"で後から追加予定です。」
- **ボタン**: 閉じる

### インタラクションゾーン

| ゾーン | タップ対象 | 期待結果 |
|--------|-----------|----------|
| 左上 | ← ボタン | S-06 へ戻る |
| データを取り出す | 行全体 | 未実装モーダル表示（MVP） |
| クラウド同期 | トグル | 未実装モーダル表示（MVP） |
| データを削除する | 行全体 | S-06b データ削除確認モーダル表示 |

### 5) SNS連携 / 共有ボタンが存在しないこと（確認項目）

- 設定画面および全画面に SNS連携・共有・公開 に該当する導線は置かない
- 「シェア」アイコン、外部投稿を想起させる文言を使用しない

**注:** この方針は既に「制約（World Bible準拠）」セクションに記載済み。
