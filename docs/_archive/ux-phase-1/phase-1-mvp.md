# Phase 1: UX-MVP (P0) — 最小UX仕様セット

---

## 目的

Phase 2（ネイティブアーキテクチャ）およびPhase 3（実装）の開始を阻害しない、最小限のUX仕様セットを完成させる。

MVP完了後、開発チームは以下を把握できる状態になる：
- 全画面構成と主要フロー
- 2.5Dワールドの構造と合成ルール
- 基本的なビジュアル・モーショントークン
- 時間帯・季節のライティング仕様
- コアとなるマイクロコピー
- アクセシビリティ基準

---

## Definition of Done

UX-MVP完了の条件：

- [ ] 35イシューすべてがクローズされている
- [ ] 以下の成果物がリポジトリに格納されている：
  - [ ] 画面フロー図（Figma + Markdown）
  - [ ] コアワイヤーフレーム（トップ、記録、棚図鑑、設定、オンボーディング）
  - [ ] デザイントークン（カラー、タイポ、スペーシング、形状）
  - [ ] モーショントークン（速度、イージング、遷移パターン）
  - [ ] 2.5Dアセット仕様書（レイヤースタック、パイプライン、合成ルール）
  - [ ] ライティング仕様書（時間帯6区分、季節4種）
  - [ ] サウンド方針書
  - [ ] コアマイクロコピー一覧
  - [ ] アクセシビリティ要件書
- [ ] Phase 2チームへの引き継ぎ準備が完了している
