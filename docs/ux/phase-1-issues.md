# Phase 1: UX Design — Issue一覧

---

## 目的

Phase 0（World Bible）で定義された世界観・哲学を、具体的なUX仕様・デザインシステム・アセット仕様へと落とし込む。

Phase 1の成果物は、Phase 2（ネイティブアーキテクチャ）およびPhase 3（実装）を阻害なく開始できる状態にすることを目標とする。

---

## 成果物

| カテゴリ | 成果物 | 形式 |
|---------|--------|------|
| 情報設計 | 画面フロー図、IA図、ワイヤーフレーム | Figma + Markdown |
| ビジュアル | カラーパレット、タイポグラフィ、トークンテーブル | Figma + JSON/Markdown |
| モーション | アニメーション仕様書、タイミングテーブル | Markdown + 動画サンプル |
| 2.5Dアセット | アセット仕様書、レイヤースタック定義、一貫性チェックリスト | Markdown + 参照画像 |
| ライティング | 時間帯/季節トーンテーブル、合成ルール | Markdown + 色見本 |
| サウンド | サウンドマップ、音量/頻度ルール | Markdown + 音源リスト |
| マイクロコピー | 全画面コピー一覧、チェックリスト | Markdown |
| アクセシビリティ | 要件定義書、チェックリスト | Markdown |
| QA | 整合性チェックリスト、レッドフラグリスト | Markdown |

---

## Definition of Done

Phase 1完了の条件：

- [ ] 全79イシューがクローズされている
- [ ] 全成果物がリポジトリに格納されている
- [ ] World Bibleとの整合性レビューが完了している
- [ ] レッドフラグ（比較圧・催促・雑然・Webby UI）がゼロである
- [ ] Phase 2チームが実装開始可能な状態である

---

## マイルストーン分類

Phase 1の79イシューは以下の3つのマイルストーンに分類される：

| Milestone | Priority | イシュー数 | 目的 |
|-----------|----------|-----------|------|
| **UX-MVP** | P0 | 35 | Phase 2/3の開始を阻害しない最小UX仕様セット |
| **UX-SOON** | P1 | 39 | MVP直後に品質を向上させる重要な拡張 |
| **UX-LATER** | P2 | 5 | 後回し可能な磨き込み・深い検証 |

### UX-MVP (P0) — Phase 2/3開始に必須

以下が含まれる：
- コアIA・フロー（トップ→記録→棚図鑑→設定）
- 2.5Dワールドアセット仕様（レイヤースタック、フォーマット、合成ルール）
- 時間帯ライティング仕様（6区分）＋季節オーバーレイ仕様（4季節）
- 最小限のビジュアルトークン（パレット、タイポグラフィ、スペーシング、形状）
- 最小限のモーショントークン（速度/イージング、遷移、記録完了セレブレーション基本）
- コアマイクロコピー（空状態、エラー、記録完了）
- 最小限のサウンド仕様（方針、儀式音）
- 最小限のアクセシビリティ基準（Dynamic Type、コントラスト、タップターゲット、Reduced Motion）

### UX-SOON (P1) — MVP直後の品質向上

以下が含まれる：
- グレードシステム詳細、検索UI、通知設定・ルール
- アイコンシステム、コンポーネントライブラリ拡充
- 世界レイヤー呼吸、マイクロインタラクション、演出詳細
- 年代・世帯別プロップ、アンビエントループ、アート一貫性チェックリスト
- トーン合成詳細、グラデーション補間
- サウンドアセット、時間帯変化、音量調整
- 通知コピー、UIラベル完全版
- 片手操作、画面サイズ対応
- QAチェックリスト

### UX-LATER (P2) — 後回し可能

以下が含まれる：
- レシピ提案ロジック
- 無音・静寂設計
- 季節・時間帯別挨拶コピー
- VoiceOver/TalkBack対応要件
- Phase 1成果物最終レビュー

## UX-001: 全体画面フロー設計
**Milestone:** UX-MVP
**Priority:** P0

**Labels:** `ux`, `ia`, `critical`

### 概要
アプリ全体の画面構成と遷移フローを定義する。トップ画面（キッチン世界）を起点とした全画面の関係性を明確にする。

### コンテキスト
World Bibleでは「トップ画面のキッチンは呼吸している」「UIは静かに、確実に使える」と定義されている。固定フッターによるナビゲーションと、世界レイヤーの分離が必要。

### 受け入れ条件
- [ ] 全画面のリストが定義されている
- [ ] 画面間の遷移関係がフロー図で可視化されている
- [ ] フッターナビゲーション構成が決定している
- [ ] 主要ユーザージャーニー（記録、探索、振り返り）が明記されている
- [ ] World Bibleの「3層構造（世界/UI/演出）」との対応が明記されている

### 依存関係
- なし（起点イシュー）

### 受け入れ条件
- [ ] 棚図鑑の画面構成が定義されている
- [ ] カードの並び順ロジックが定義されている（作成日順/グレード順/ジャンル別等）

- [ ] スクロール方式（縦/横/無限）が決定している

## UX-003: 料理カード構造設計
**Milestone:** UX-MVP
**Priority:** P0

**Labels:** `ux`, `ia`

### 概要
棚図鑑に並ぶ「料理カード（recipe card）」の情報構造を設計する。1カード=1料理、同じ料理はグレードアップで表現。

### コンテキスト
World Bible: 「同じ料理を作ってもカードは増えず、代わりにグレードが上がる。」「写真は棚図鑑に紐づく料理カードの一部として積み重なる。」

### 受け入れ条件
- [ ] カードに含まれる情報要素が定義されている（料理名、写真、グレード、作成回数、最終作成日等）
- [ ] カードの視覚的階層（何を目立たせるか）が定義されている
- [ ] カードのタップ時挙動が定義されている
- [ ] カードのサイズバリエーション（一覧/詳細）が定義されている

### 依存関係

- [ ] グレードアップ時のUXフローが定義されている

## UX-006: 写真追加フロー設計
**Milestone:** UX-SOON
**Priority:** P1

**Labels:** `ux`, `ia`

### 概要

- [ ] 複数写真の扱いが定義されている
